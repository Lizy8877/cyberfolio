Fondamentaux de l'algorithmie et Programmation Python

Contexte et description du projet

Ce projet a pour but de mettre en pratique les notions de programmation en Python vues en première partie du module. Il se fera en 3 temps.

Premièrement, les étudiants vont développer une application qui permet à un commerçant de gérer une liste de ses produits. L'application permettra à l'utilisateur (le commerçant en question) de lire un fichier texte (.txt) contenant une liste de produits et d'effectuer des opérations comme l'ajout, la recherche et le tri de ses produits. Le code doit être structuré de façon à ce qu'il soit clair, efficace et modulaire.

Deuxièmement, les étudiants devront étendre la première partie du projet à une plateforme de gestion de commerçants. Pour ce faire, il faudra une base de données locale sous la forme d'un fichier .csv contenant les utilisateurs ainsi que leurs mot de passe. Chacun aura sa liste de produits à gérer. Il faudra ensuite renforcer la sécurité de la base de données. Il va également falloir détecter les mots de passe compromis et proposer des suggestions pour les renforcer. Les étudiants généraliseront la détection des mots de passe ou des adresses mail compromis en utilisant une API. Votre gestionnaire devra alors envoyer des alertes aux utilisateurs concernés.

Troisièmement, munir votre gestionnaire d'une interface graphique en intégrant un outil comme Tkinter ou PyQt.

La roadmap du projet et les différents jalons

Le projet s'est effectué en 6 jalons :

  1. Menu de choix et fonctionnalités de base :
    • Un menu interactif pour naviguer entre les différentes options.
    • Lecture et écriture dans un fichier texte pour sauvegarder et charger les données.
    • Utilisation de la programmation modulaire pour structurer le code.
    • Permettre d'ajouter de nouveaux produits ou de supprimer des produits de la liste.
    • Permettre de rechercher un produit par son nom. En plus de la recherche séquentielle, il vous est demandé de développer la recherche binaire.
    • Permettre d'afficher la liste des produits, triée par prix ou par quantité. Vous devrez développer les algorithmes de tri par vous même. Il vous est demandé de développer au moins deux algorithmes (par exemple : tri à bulles et tri rapide).


  2. Adapter les fonctionnalités à un fichier CSV :
    • Implémentez une base de données pour stocker les utilisateurs et leurs mots de passe sous forme de fichier CSV.
    • Permettez d'ajouter de nouveaux utilisateurs et de mettre à jour ou supprimer leurs informations dans la base de données.
    • Générer une base de données de hachages compromis.
    • Faire transition d’un fichier texte à une base de données pour les produits. Vous pouvez désormais utiliser un module comme pandas pour la recherche et le tri.


  3. Sécurisation de la base de données :
    • Générer un hashage approprié des mots de passe des utilisateurs avec hashlib.
    • Comparer ces hashages avec la base de données des hashages compromis.
    • Indiquer quels mots de passe sont compromis et proposer des suggestions pour renforcer les mots de passe.
    • Implémenter un système de génération de salage des mots de passe avant de les hasher.
    • Générer une mise à jour de la base de données avec un hashage adéquat.


  4. Détection des compromis en utilisant une API :
    • Implémenter la vérification des mots de passe et des adresses e-mail via l'API « Have I Been Pwned » pour chaque élément.
    • Générer une base de données locale avec l'historique des requêtes et des compromis.


  5. Système de notifications en cas de données compromises :
    • Ajouter un système de notification ou d'alerte par envoi de mail en cas de nouveaux compromis détectés pour un utilisateur.


  6. Interface graphique simple :
    • Remplacer le menu console par une simple interface graphique.
    • Permettre de voir ses produits et faire une recherche.

Les objectifs pédagogiques et professionnels du projet

Les savoirs

  • Comprendre les structure de données en Python (listes et dictionnaires).
  • Apprendre à manipuler des fichier texte et CSV en Python.
  • Apprendre à manipuler des bases de données locales en Python.
  • Comprendre les concepts avancés de hashage (salage, force brute).
  • Comprendre l'importance de la gestion des mots de passe et des compromis de sécurité.
  • Comprendre les API REST et apprendre à interagir avec des API tierces.
  • Apprendre à envoyer des emails de notification en Python.

Le savoir-être

  • Respecter les délais.
  • Évaluer sa propre contribution.
  • Strucutrer le code de manière claire et modulaire pour faciliter la maintenance et l'évolution.
  • Travailler de manière organisée pour gérer plusieurs fonctionnalités.
  • Assurer une gestion responsable et sécurisée des mots de passe utilisateurs.

Les compétences

  • Développer une application en Python modulaire et fonctionnelle.
  • Utiliser des strucutres de contrôle pour gérer des choix utilisateurs via un menu (qu'il soit graphique ou non).
  • Implémenter des processus de salage et de hashage puis comparer les hashages avec une base de données compromises.
  • Développer une application Python qui interagit avec une API externe et stocke les résultats dans une base de données.
  • Mettre en place un système de surveillance pour informer les utilisateurs de nouveaux compromis.







La réalisation du projet d'Algorithmie

Le projet d'Algorithmie a été réalisé en binôme sur une durée de 3 semaines. Chaque semaine, chaque groupe devait rendre son code afin de visualiser l'avancée des projets de chaque binôme.
Comme expliqué dans le cahier des charges, chaque semaine, nous devions incorporer des fonctionnalités à notre application afin de la faire évoluer. Dans le cahier des charges (que vous retrouverez ci-dessous), il était indiqué que nous devions faire en sorte d'ajouter la possibilité de passer des commandes ainsi que la possibilité de voir les statistiques. Par manque de temps, cette partie-là a été abandonnée pour l'ensemble des groupes (avec accord de l'intervenant).

Pour réaliser notre projet, nous avons opté pour l'utilisation de l'API "Have I Been Pwned" lors de la création du compte. Cela sécurise davantage l'application car il est alors impossible pour l'utilisateur de choisir un mot de passe compromis. Nous avons utilisé le fichier CSV rockyou.csv afin de sécuriser la partie "Gestion du compte" dans le cas où l'utilisateur souhaite modifier son mot de passe. Pour davantage de sécurité sur les mots de passe, une comparaison à l'API "Have I Been Pwned" a été ajoutée lors de la connexion. En effet, la liste des mots de passe compromis évolue de jour en jour et un mot de passe qui n'était pas compromis aujourd'hui pourrait bien l'être demain. Dans le cas d'un mot de passe compromis détecté, un email est automatiquement envoyé à l'utilisateur pour le prévenir de la compromission de son mot de passe et l'incite à en changer.

De plus, la partie afin de vérifier la compromission des adresses mails n'a pas été implémentée car il s'agit d'une fonctionnalité payante de l'API "Have I Been Pwned".
