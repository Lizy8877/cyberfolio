Programmation web full stack

Contexte et description du projet

Le contexte du projet

Toute formation consiste en un parcours pédagogique permettant d'atteindre un objectif professionnel. Ce parcours pédagogique donne lieu à l'acquisition d'un ensemble de compétences qui doivent être mises en valeur par l'étudiant à l'issu de sa formation afin de maximiser son employabilité. Destinés à de potentiels recruteur, l'ensemble des compétences acquises, ou plus simplement "portfolio" doit, à l'ère du numérique, être accessible en ligne. Ainsi, quoi de plus évident pour un développeur web full stack que de concevoir et développer lui-même son cyberfolio ?


Les objectifs du projet

Ce projet a pour principaux objectifs d'appréhender l'architecture client-serveur et de mettre en opeuvre de manière simple les langages les plus utilisés dans le développement web full stack :

  • Côté client, ou "frontend" : HTML, CSS et JavaScript.

  • Côté serveur, ou "backend" : PHP, MySQL, Python, langage C.

Parmi les objectifs secondaires, les étudiants seront amenés à installer, configurer et prendre en main un environnement de travail local (éditeur de code et serveur web local), à cerner avec précision ce en quoi consiste le métier de développeur web et à concevoir leur futur cyberfolio. Ce projet sera également l'occasion de découvrir un premier aspect relatif à la sécurité des applications web : le hashage des mots de passe.

La roadmap du projet et les différents jalons

Le projet s'est effectué en 8 jalons :

  1. Création de la fiche de persona du développeur web full stack.
  2. Réalisation des pages en HTML et CSS en version statique pour ordinateur et pour mobile.
  3. Réalisation du site en dynamique avec l'inclusion de la base de données pour les pages de projets.
  4. Création des pages d'inscriptions et de login.
  5. Ajout d'une table de données pour les utilisateurs.
  6. Implementation de l'API "Have I Been Pwned" afin de connaître la dernière faille de sécurité.
  7. Implémentation de la fonction de hashage des mots de passe.
  8. Présentation orale du projet.

La progression du projet

Lors de la semaine 1, Il s'agissait de découvrir le métier de développeur web ainsi que du développement frontend :

  • Recherches, analyses et synthèse des compétences techniques et transversales du métier de développeur web full stack.

  • Élaboration d'un persona "développeur web full stack" représentatif du groupe du projet.

  • Définition du contenu et de la structure (arborescence) du cyberfolio.

  • Découverte du langage frondend : HTML et CSS (le JavaScript est facultatif).

  • Prise en main de l'éditeur de code : Visual Studio Code.

  • Création d'une première version statique du cyberfolio en HTML et CSS.

  • Création de la version responsive du site.

Au cours de la semaine 2, il s'agissait de faire le développement backend du site :

  • Installation et configuration d'un serveur web local (via la solution UwAmp).

  • Élaboration d'unpremier script en PHP illustrant les bases syntaxiques du langage.

  • Déploiement d'une base de données sur le serveur MySQL

  • Découverte des bases syntaxiques du langage MySQL à travers des requêtes d'interrogations simples (SELECT).

  • Création d'une table MySQL destinée à stocker les informations principales du cyberfolio (nom, prénom, les projets, etc.).

  • Création d'un script PHP de connexion à la base de données MySQL via PDO.

  • Création d'une seconde version du cyberfolio (dynamique) en HTML, CSS et PHP, intégrant les données issues de la table MySQL préalablement créée.

  • Version responsive du cyberfolio.

Lors de la dernière semaine du projet, il a fallu sécuriser le cyberfolio créé :

  • Création d'une page d'inscription.

  • Création d'une page de connexion.

  • Recherche et réflexions autour du mécanisme de session, de la gestion de l'authentification et des autorisations, et des principales vulnérabilités d'une application web.

  • Affichage en JavaScript sur la page de connexion de la dernière faille de sécurité recensée via l'API RESTful "Have I Been Pwned".

  • Intégration d'un outil de détection de fonction de hashage utilisée pour hasher les mots de passe des utilisateurs.
  • La version statique !
  • La version dynamique !
